# Sources

Pokopia Habitat Planner is an unofficial fan-made planning tool for Pokémon Pokopia.

The app uses publicly available information from Serebii.net and organizes it into planning views for habitat compatibility, item coverage, acquisition, recipes, material sources, and material workflows.

This project does not claim ownership over Pokémon, Pokémon Pokopia, Pokémon names, item names, sprites, images, or related assets.

## Main data source

Most game data used by the tool comes from Serebii’s Pokémon Pokopia pages:

https://www.serebii.net/pokemonpokopia/

## Favorite category pages

The Habitat Designer, item coverage system, Pokémon compatibility, and shared category recommendations use the Pokémon Pokopia favorite category pages from Serebii.

These pages are used to identify:

- Pokémon favorite categories
- item favorite categories
- Pokémon habitat preferences
- Pokémon specialties
- comfort/category overlap between Pokémon and items

## Items, acquisition, recipes, and materials

The app also uses Serebii’s item, crafting, and material-source pages.

These pages are used to identify:

- item descriptions
- item categories
- item acquisition methods
- recipe/crafting information
- material usage
- material sources
- Litter/material-producing Pokémon
- curated material workflows

- Crafting: https://www.serebii.net/pokemonpokopia/crafting.shtml

### Material source data

## Images and sprites

Pokopia Habitat Planner may locally cache Pokémon and item images from publicly available Serebii pages to help users visually identify Pokémon and items inside the planner.

The optional image cache included in some beta releases is provided only to improve loading convenience for testers.

All Pokémon, item images, sprites, names, and related visual assets belong to their respective owners.

## Manual / curated rules

Some material workflow rules are curated manually from Serebii item and crafting descriptions to avoid incorrect automatic parsing.

For example, the app does not infer conversion rules broadly from any item name found in text. Instead, conversion workflows are kept in a curated workflow file so that material logic can be checked and corrected.

The curated workflow file is used to avoid false relationships such as treating decorative objects or location names as crafting stations.
